═══════════════════════════════════════════════════════════════
                      PRIORITY ALGORITHM
═══════════════════════════════════════════════════════════════

HIGH PRIORITY (🟢):
├─ Skills match: 80%+
├─ Experience fit: Perfect for junior with 2 years
├─ Tech stack: Your exact technologies
├─ Career path: Aligns with AI/full-stack goals
└─ Example: "Junior Full Stack - React, Python, AI focus"

MEDIUM PRIORITY (🟡):
├─ Skills match: 50-79%
├─ Experience fit: Reasonable with some stretching
├─ Tech stack: Most technologies, some gaps
├─ Career path: Good direction, minor detours
└─ Example: "Junior Developer - React focus, nice-to-have Python"

LOW PRIORITY (🔴):
├─ Skills match: <50%
├─ Experience fit: Too junior or too senior
├─ Tech stack: Different technologies
├─ Career path: Different direction
└─ Example: "Senior Backend Engineer - 5+ years, C++/Rust"

                       EXECUTION MODES
═══════════════════════════════════════════════════════════════

MODE 1: RUN ONCE
Command: python main.py --once
When: Manual job search
Result: One-time HTML generation
Use: When you want to check jobs now

MODE 2: SCHEDULED
Command: python main.py --schedule
When: Automatic daily updates
Result: Runs every 24 hours
Use: Set and forget, always fresh jobs

MODE 3: DOCKER
Command: docker-compose up -d
When: Background 24/7 service
Result: Containerized automation
Use: Production deployment
